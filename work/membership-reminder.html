overview
the idea.

most people leak money to forgotten subscriptions — the gym they cancelled, the streaming trial they meant to undo, the cloud plan that auto-doubled.

tegel is a small desktop app that keeps a single source of truth for every membership. enter the name, cost, and renewal date once; it tracks the rest. a quiet notification fires three days before each renewal so cancelling is never a surprise.

built solo for an object-oriented programming course as a way to push myself past the usual “to-do list” coursework example.

what i built

small app, full surface.

01

membership crud

add, edit, and remove memberships with a clean list view. each row is editable inline; the sidebar holds add/edit/delete entry points and a search field for long lists.

javafx fxml
02

spend chart

a “total pengeluaran” chart aggregates monthly spend across all active memberships. the y-axis auto-scales so a single $5 service still draws something useful.

javafx chart
03

renewal reminders

a background scheduler checks every membership on app launch and at midnight. anything renewing in the next three days fires a desktop notification and lights up the sidebar bell.

scheduler notifications
04

local-only data

everything sits in a sqlite file in the user's home directory — no servers, no accounts. since this is personal financial data, that felt like the right default.

sqlite jdbc
