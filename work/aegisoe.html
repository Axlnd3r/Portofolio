overview
the problem.

secrets leak. teams push api keys, db passwords, and signing keys to public repos every day — usually without realizing it. by the time a human notices, the credential is already scraped.

aegisoe closes the loop. it watches github, classifies risk with three independent signals, then rotates the credential automatically inside a trusted execution environment so the new value is never exposed in plaintext to anyone — including the operator.

every incident is anchored on‑chain with a sha-256 commitment, so auditors can prove a leak was caught and resolved without seeing the secret itself.

how it works

three layers, one verdict.

01

regex pattern matching

fast first pass over diffs and pushes — looks for known credential shapes (aws keys, github tokens, jwt signatures, private keys). cheap and high recall.

github webhook regex
02

shannon entropy analysis

second pass catches unknown-shape secrets by flagging high-entropy strings inside source files. tunable threshold per file type to avoid hashing minified js.

entropy scoring
03

llm risk classification

final reasoning pass — a language model reads the surrounding code to judge intent (test fixture vs production credential) and assigns a risk grade. sharply reduces false positives.

llm classification
04

confidential rotation

chainlink cre executes the rotation inside a trusted execution environment. neither the operator nor the network sees the new credential — it's pushed straight to the customer's secret store.

chainlink cre tee
05

on-chain proof

each incident emits a sha-256 commitment to a sepolia contract — repo id, timestamp, classification, resolution. fully verifiable, fully immutable, fully transparent — without ever revealing the secret itself.
